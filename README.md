# QA-project-9
Учебный проект по безопасности веб-приложений: разбор угроз, инструменты криптографии, тест-план безопасности, сценарии для забытой сессии.

### Задание 1. Виды безопасности

Представь, что ваша группа - команда QA, которая начинает тестирование новой функциональности в веб-приложении — "Безопасный чат для врачей и пациентов". В чате передаются конфиденциальные диагнозы, назначения и личные данные.
Учитывая высокую критичность данных (персональные данные и медицинская тайна), необходимо провести предварительный анализ угроз безопасности.

Ваша задача: обсудить возможную функциональность приложения и на этой основе выявить потенциальные уязвимости и риски, нарушающие принципы защищенного приложения, и классифицировать их по основным причинам проблем безопасности.
1. Составь список из 3-5 потенциальных рисков/сценариев нарушения принципа конфиденциальности, где данные чата могут стать доступны посторонним. К каждому риску укажи причину проблемы безопасности. 
2. Составь список из 3-5 потенциальных рисков/сценариев нарушения принципа целостности, где сообщения или файлы в чате могут быть изменены без ведома участников. К каждому риску укажи причину проблемы безопасности. 
3. Составь список из 3-5 потенциальных рисков/сценариев нарушения принципа доступности, при которых легитимный пользователь (врач или пациент) не сможет получить доступ к чату в критический момент. К каждому риску укажи причину проблемы безопасности.
3. Оформите ваши идеи в виде отчета. Отчет должен содержать:
    - Введение: краткое описание анализируемого приложения и функциональности.
    - Матрица рисков: таблица с объединенными рисками для каждого из принципов.
    - Топ-3 критических риска по мнению вашей группы (совместно выберите три самых опасных риска из всего списка и обоснуйте свой выбор)
    - Рекомендации для разработчиков: для каждого из топ-3 рисков предложите 1-2 конкретные меры по их устранению или снижению (митигации). 
4. Выполненное задание (отчет) оформите в документе `task_1.md`.

### Задание 2. Инструменты криптографии

1. Самостоятельно изучи теорию и опиши разницу между кодированием, хешированием и шифрованием. Приведите по 1 реальному примеру использования каждого из них в веб-приложении.
2. Представь, что тебе нужно объяснить бизнесу (менеджеру проекта) разницу между симметричным и асимметричным шифрованием, чтобы менеджер, который не является экспертом в безопасности, понял тебя. Придумай аналогию, подготовь краткую сводку по симметричному и асимметричному шифрованию (определение, аналогия, плюсы и минусы). 
3. Составь "черный" и "белый" списки алгоритмов симметричного шифрования и хеширования.

Для “черного” списка укажи по  2-3 слабых алгоритма шифрования и хеширования и опиши конкретные типы атак, к которым они уязвимы.

Для “белого” списка укажи по 1-2 рекомендованных алгоритмов (стойких к различным атакам) шифрования и хеширования и объясни, почему они считаются безопасными.

4. Представьте, что вы команда тестировщиков в компании, которая разрабатывает интернет-магазин электроники. 
Ваша задача - составить тест-план для проверки базовой безопасности приложения. Фокус - ручное тестирование. 

Выберите 3 области для проверки из списка ниже:

  - Вход в аккаунт (логин/пароль)
  - Работа с корзиной и заказами
  - Личный кабинет пользователя
  - Поиск товаров
  - Оформление заказа

Для каждой выбранной области опишите, что и как будете проверять, на что обращать внимание. 

Опишите, какие web инструменты помогут тестировщику для проведения проверок. 

Опишите проверку для возможной ситуации: “Пользователь забыл разлогиниться на общественном компьютере”: 
  - *Как это можно проверить?*
  - *Что должно происходить для защиты?*

5. Выполненное задание (сводка, списки, тест-план и проверка для ситуации) оформите в документе `task_2.md`.
